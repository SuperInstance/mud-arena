## Audit Findings from AUDIT-REPORT.md

12 models audited the CUDA kernel. Key issues found:

| Issue | Severity | Status |
|-------|----------|--------|
| Race conditions in shared memory (s_room) | **CRITICAL** | Needs fix |
| No agent state write-back to global memory | **CRITICAL** | Needs fix |
| Missing `__syncthreads()` after agent actions | **HIGH** | Needs fix |
| Uncoalesced global memory access | **MEDIUM** | Optimization |
| No error handling for CUDA errors | **MEDIUM** | Improvement |

The existing audit is already organized and actionable. Good practice.

---

## Recommendations

1. **Fix the CRITICAL CUDA issues** — race conditions and write-back. These are correctness bugs, not just performance.
2. **Promote the AUDIT-REPORT.md pattern** — other repos should have similar audit docs.
3. **Add AUDIT-REPORT.md to the README** — make it visible that the codebase has been audited.
4. **Add a CONTRIBUTING.md** — the boarding manifesto is great but there's no formal PR process.
5. **Consider a `Makefile` target for audit** — `make audit` to run the 12-model audit pipeline.
